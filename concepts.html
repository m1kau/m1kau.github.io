Key Concepts

These core ideas explain how browsers find, fetch, interpret, and display the web.

URL (Uniform Resource Locator)

A URL is the address for anything on the web. It tells the browser what protocol to use, which host to contact, and which resource to request.

- Protocol: https:// vs http:// (security + transport)

- Host: domain or IP (e.g., example.com)

- Path & query: which file or API and optional parameters

HTTP / HTTPS

HTTP is the request response language of the web. HTTPS is HTTP with encryption and identity verification, which protects data from eavesdropping and tampering.

- Request/Response: browser asks, server answers (status codes like 200, 404)

- HTTPS: TLS encryption + certificates for trust

- Modern features: HTTP/2 multiplexing and HTTP/3 over QUIC

HTML (Structure)

HTML defines the structure and meaning of content: headings, paragraphs, links, images, sections, tables, and more.

- Semantic tags: <header>, <main>, <article>

- Accessibility: proper structure helps screen readers

- DOM: browsers parse HTML into a live document tree

CSS (Presentation)

CSS controls the look and layout: colors, fonts, spacing, and responsive design. It lets the same HTML adapt to phones, tablets, and desktops.

- Selectors: target elements by tag, class, id, state

- Layout: Flexbox and Grid for modern page structure

- Responsive: media queries adjust designs at breakpoints

JavaScript (Behavior)

JavaScript adds interactivity and app-like behavior. It can read and change the DOM, respond to events, and talk to servers.

- DOM APIs: manipulate content and styles dynamically

- Events: clicks, input, network results drive UI updates

- Modules & fetch: structure code and load data

Rendering Engine & JS Engine

The rendering engine turns HTML/CSS into pixels, while the JavaScript engine executes code that powers interactivity.

- Pipeline: parse → style → layout → paint → composite

- Engines: Blink/WebKit (rendering), V8/SpiderMonkey (JS)

- Performance: minimizing reflow/repaint keeps pages smooth

Process Model & Sandboxing

Modern browsers isolate tabs and sites into separate processes. This boosts stability and security—one crash or malicious page won't take down everything.

- Isolation: per-tab or per-site processes

- Sandbox: limits what web content can access

- Stability: a bad tab doesn't crash the whole browser

WebAssembly (Wasm)

WebAssembly lets browsers run compiled code at near-native speed. It powers heavy apps like video editors and games, alongside JavaScript.

- Performance: fast, compact binary format

- Interop: works with JS and the web platform

- Use cases: compute-intensive apps directly in the browser
